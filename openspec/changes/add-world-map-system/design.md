## Context

- 当前 `Character` 已包含 `Position(x, y)`，并用于探测距离计算。
- 当前 UI 没有地图视图；坐标不在主界面中出现。
- 当前主循环会持续执行 AI 决策和世界更新，适合接入地图上下文。

## Goals / Non-Goals

**Goals:**
- 给世界引入最小但完整的地图领域模型
- 保证角色坐标在地图边界内
- 在 UI 中可视化角色位置
- 保持现有架构与可运行性

**Non-Goals:**
- 不实现复杂路径寻路/障碍物系统
- 不实现分块地图或多地图切换
- 不实现小地图交互操作

## Decisions

### 1. 地图建模放在 `world` 域

**决策：** 新增 `WorldMap`，包含 `width/height/min_x/min_y` 与以下方法：
- `clamp_position`: 边界裁剪
- `normalize`: UI 映射到 [0,1]
- `to_dict`: 输出到环境上下文

**理由：** 地图是全局世界状态，不应放在 UI 或角色域。

### 2. 在 `World` 层统一做位置合法化

**决策：** 在 `add_character` 与 `update_character` 时自动裁剪坐标。

**理由：** 保证任何来源（初始化、AI 更新、读档）最终进入世界状态时都合法。

### 3. AI 位置变化采用“可选扩展”

**决策：** 在 `CharacterChange` 增加 `position`，支持：
- 绝对坐标：`x`, `y`
- 位移：`dx`, `dy`

**理由：** 与现有 JSON 格式兼容，不要求 AI 必须输出位置字段。

### 4. UI 地图采用轻量二维点图

**决策：** 左下角新增地图面板，绘制网格 + 角色点位 + 名称。

**理由：** 第一版重点是可见性与信息价值，不引入复杂渲染逻辑。

## Risks / Trade-offs

| 风险 | 影响 | 缓解 |
|---|---|---|
| AI 输出非法坐标 | 中 | 统一由 `World` 进行边界裁剪 |
| UI 空间变紧凑 | 低 | 调整左下区域为地图+物品栏分区 |
| 后续地图玩法扩展受限 | 中 | `WorldMap` 独立建模，后续可追加地形/区域字段 |
