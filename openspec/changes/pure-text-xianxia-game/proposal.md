## Why

创建一个以 AI 驱动的纯文字修仙模拟游戏，通过大语言模型模拟修仙世界中所有角色的自主行为和交互，实现一个动态、沉浸式的修仙世界体验。

## What Changes

- 初始化项目结构，使用 pygame 作为游戏框架，uv 作为包管理器
- 实现角色属性系统（血量、灵力、神识、境界、灵根等）
- 实现物品系统（法宝、丹药、灵石等）
- 实现记忆系统，记录角色的历史经历和目标
- 实现 AI 交互系统，将角色信息打包为提示词并解析 AI 返回的格式化数据
- 实现角色探测和交互系统，处理多角色同步场景
- 实现境界突破系统（小境界和大境界）
- 实现战斗伤害计算系统
- 实现数据库管理系统，存储法宝、功法等游戏数据

## Capabilities

### New Capabilities
- `character-system`: 角色属性、境界、灵根、记忆等核心角色系统
- `item-system`: 法宝、丹药、物品栏等物品管理系统
- `ai-interaction`: AI 提示词构建、响应解析、属性变化处理
- `combat-system`: 战斗伤害计算、法宝损耗、状态效果
- `realm-system`: 境界突破、概率计算、失败惩罚
- `detection-system`: 角色探测范围、多角色交互同步
- `world-database`: 游戏数据存储、检索、动态创建

### Modified Capabilities
- 无

## Impact

- 新建完整的 Python 项目结构
- 依赖 pygame 进行 UI 渲染
- 依赖大语言模型 API 进行角色行为模拟
- 使用配置文件管理世界观参数
- 数据库用于存储法宝、功法等游戏数据
